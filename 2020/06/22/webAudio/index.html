解析音频

读取指定的 Blob 或 File 内容 写法与 new Image() 类似，最后返回一个 ArrayBuffer

createBuffer() 阻塞了其他代码的执行。异步方法 decodeAudioData() 能够完成相同的工作—— 传入一个压缩过的音频（如MP3格式的文件），并直接返回一个可以通过 AudioBufferSourceNode 播放的 AudioBuffer 。
播放诸如MP3等格式的压缩音频时，你应当使用 decodeAudioData() 方法

解析最后得到的是一个 AudioBuffer 对象

音频可视化

截取拼接视频波段核心方法 getBufferBetween，concatwebAudioBuffers

getBufferBetween(start, end)
传入截取的首尾X两个坐标，通过等比计算出这个区间所在的当前视频的时段，如果不做可视化就不需要做什么计算了，直接传时间段就好了比如：0~2.1秒;
AudioBuffer 接口的getChannelData()方法返回一Float32Array ，其中包含与通道关联的PCM数据，通道参数定义(0表示第一个通道);
AudioContext的sampleRate属性返回一个浮点数表示采样率（每秒采样数）， 同一个AudioContext中的所有节点采样率相同，所以不支持采样率转换;

concatwebAudioBuffers(buffer1, buffer2)
传入之前拼接的音频buffer 与 当前新选择截取的音频Buffer；
AudioBuffer对象可以通过 AudioContext.createBuffer 来创建或者通过 AudioContext.decodeAudioData成功解码音轨后获取；
AudioBuffer接口的numberOfChannels属性返回一个整数，该整数表示由缓冲区中存储的PCM数据描述的离散音频通道的数量；
把截取的音频串起来，buffer1.length + buffer2.length。
